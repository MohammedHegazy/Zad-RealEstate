# الفصــل الثامن

# الأدوات واللغات المستخدمة

## 8.1. مقدمة

يعرض هذا الفصل الأدوات واللغات والتقنيات المستخدمة في تنفيذ مشروع زاد للعقارات. تم اختيار هذه الأدوات بما يناسب طبيعة النظام، لأنه يعتمد على واجهة أمامية مستقلة، خادم API، قاعدة بيانات، وخدمة تعلم آلي للتنبؤ بسعر العقار.

يعتمد المشروع على Vue في الواجهة الأمامية، Laravel في الواجهة الخلفية، وFlask مع scikit-learn في خدمة التنبؤ السعري. كما يستخدم أدوات مساعدة لإدارة الحزم، تشغيل الخوادم، بناء الواجهة، اختبار النظام، وتوثيق قاعدة البيانات والمخططات.

## 8.2. النموذج المستخدم في تطوير المشروع

اعتمد المشروع على أسلوب تطوير تكراري، حيث تم بناء الوظائف على مراحل: بداية من قاعدة البيانات والمصادقة، ثم العقارات والواجهات العامة، ثم الإدارة، ثم التوصيات والاستثمار والثقة والتنبؤ السعري. يساعد هذا الأسلوب على اختبار كل جزء قبل الانتقال إلى جزء أكبر.

أما من الناحية المعمارية، فيعتمد النظام على نموذج Client/API. الواجهة الأمامية لا تتصل بقاعدة البيانات مباشرة، بل ترسل طلبات إلى Laravel API. ويقوم Laravel بتنفيذ منطق الأعمال والتعامل مع قاعدة البيانات والخدمات الداخلية. وعند الحاجة إلى توقع سعر عقار، يرسل Laravel البيانات إلى خدمة Flask المنفصلة.

كما يستخدم Laravel نموذج MVC، حيث تمثل النماذج البيانات والعلاقات، وتتعامل المتحكمات مع الطلبات، وتنفذ الخدمات منطق الأعمال الأكثر تعقيداً.

## 8.3. بيئة التطوير

تتكون بيئة التطوير من عدة أجزاء:

| الجزء | التقنية المستخدمة |
|---|---|
| لغة الخلفية | PHP |
| إطار الخلفية | Laravel 12 |
| المصادقة | Laravel Sanctum |
| لغة الواجهة | JavaScript |
| إطار الواجهة | Vue |
| أداة البناء | Vite |
| إدارة حالة الواجهة | Pinia |
| التنقل | Vue Router |
| الخرائط | Leaflet + OpenStreetMap |
| خدمة ML | Python + Flask + scikit-learn |
| قاعدة البيانات | MySQL أو SQLite حسب البيئة |
| الاختبارات | PHPUnit وLaravel Test Runner |
| التوثيق | Markdown وDBML |

هذا التنوع في الأدوات يعكس طبيعة المشروع، لأنه ليس تطبيقاً بسيطاً من صفحة واحدة، بل نظام متعدد الطبقات والوظائف.

## 8.4. الأدوات المستخدمة

تم استخدام مجموعة من الأدوات أثناء التطوير:

- Visual Studio Code لتحرير ملفات المشروع.
- Composer لإدارة حزم PHP وLaravel.
- Node.js وnpm لإدارة حزم الواجهة الأمامية.
- Vite لتشغيل وبناء مشروع Vue.
- Laravel Artisan لتنفيذ أوامر Laravel مثل تشغيل الخادم، الهجرة، والاختبارات.
- Git لإدارة النسخ والتغييرات.
- Postman أو أدوات مشابهة لاختبار واجهات API.
- DBML وdbdiagram لتوثيق مخطط قاعدة البيانات.
- Mermaid لتوثيق بعض المخططات داخل ملفات Markdown.
- PHPUnit لاختبار وظائف Laravel.
- Laravel Pint لتنسيق كود PHP عند الحاجة.
- Python pip لإدارة حزم خدمة التنبؤ السعري.

كل أداة تخدم جزءاً محدداً من دورة التطوير، من كتابة الكود إلى تشغيله واختباره وتوثيقه.

## 8.5. اللغات المستخدمة في المشروع

استخدم المشروع عدة لغات، ولكل لغة دور واضح:

| اللغة | الاستخدام |

يساعد هذا التوزيع على استخدام اللغة المناسبة لكل جزء، بدلاً من إجبار النظام كله على تقنية واحدة.

## 8.6. تقنيات ومكتبات الخادم الخلفي Backend

يعتمد الخادم الخلفي على Laravel 12، وهو المسؤول عن توفير واجهات API وإدارة منطق الأعمال. يستخدم المشروع Laravel Sanctum لحماية المسارات الخاصة، وEloquent ORM للتعامل مع قاعدة البيانات من خلال النماذج والعلاقات.

كما يستخدم المشروع Migrations لإنشاء الجداول وتعديلها، وFactories وSeeders لتجهيز بيانات اختبارية أو أولية. وتستخدم Form Requests للتحقق من البيانات القادمة من المستخدم، بينما تم فصل المنطق الأكبر في Services مثل خدمات التوصيات، الاستثمار، المحافظ، الثقة، البحث الجغرافي، رفع الملفات، والتنبؤ السعري.

ومن التقنيات الخلفية المهمة:

- Middleware لحماية المسارات وتنظيم الوصول.
- Resources لتنسيق استجابات JSON.
- Traits لتجميع منطق مشترك عند الحاجة.
- Laravel Filesystem لإدارة الملفات والوسائط.
- PHPUnit وLaravel Test Runner لاختبار المسارات والخدمات.

## 8.7. تقنيات ومكتبات الواجهة الأمامية Frontend

تم بناء الواجهة الأمامية باستخدام Vue، وتم تشغيلها وبناؤها باستخدام Vite. يستخدم المشروع Vue Router لتنظيم التنقل بين الصفحات العامة والمحمية والإدارية، ويستخدم Pinia لإدارة حالة المستخدم والبيانات المشتركة.

يعتمد التصميم على Bootstrap وبعض المكونات المشتركة لتوحيد شكل الواجهة. كما تم استخدام Leaflet لعرض الخرائط ومواقع العقارات. وتحتوي الواجهة على ملفات API منظمة للتعامل مع Laravel، بحيث لا يتم توزيع طلبات HTTP بشكل عشوائي داخل الصفحات.

تشمل الواجهة صفحات عامة مثل العقارات والمدن والشركات والوسطاء، وصفحات محمية مثل الملف الشخصي والمفضلة والتوصيات والاستثمار، وصفحات إدارية لإدارة المستخدمين والعقارات والشركات والوسطاء والثقة.

## 8.8. الخدمات الخارجية والمساندة

أهم خدمة مساندة في المشروع هي الخرائط، حيث يستخدم النظام Leaflet مع OpenStreetMap لعرض مواقع العقارات والبحث الجغرافي. هذه الخدمة مهمة لأن الموقع عامل أساسي في قرار شراء أو استثمار العقار.

كما توجد خدمة Flask للتنبؤ السعري، وهي خدمة محلية منفصلة عن Laravel. تستقبل هذه الخدمة بيانات العقار، تمررها إلى نموذج scikit-learn، ثم تعيد السعر المتوقع. يساعد فصل هذه الخدمة على تطوير نموذج الذكاء الاصطناعي دون التأثير المباشر على بنية Laravel.

يمكن مستقبلاً إضافة خدمات خارجية أخرى مثل الدفع الإلكتروني، البريد، الرسائل النصية، التخزين السحابي، أو خدمات الخرائط المتقدمة.

## 8.9. قاعدة البيانات والتخزين

يعتمد المشروع على قاعدة بيانات علائقية يمكن تشغيلها باستخدام MySQL أو SQLite حسب البيئة. يتم إنشاء الجداول من خلال Laravel Migrations، بينما يوضح ملف `ERD2.dbml` مخطط العلاقات بين الجداول.

تتضمن قاعدة البيانات جداول للمستخدمين، المدن، المناطق، الشركات، الوسطاء، العقارات، الصور، الفيديوهات، المفضلة، الرسائل، الإشعارات، التحليلات الاستثمارية، المحافظ، التوصيات، التفضيلات، التقييمات، طلبات التوثيق، والتنبؤات السعرية.

أما الملفات والوسائط مثل صور وفيديوهات العقارات، فيتم التعامل معها من خلال Laravel Filesystem وخدمة رفع ملفات مخصصة، مما يجعل إدارة الوسائط منفصلة عن منطق العقارات.

## 8.10. أدوات التحقق والاختبار

تم استخدام PHPUnit وLaravel Test Runner لاختبار أجزاء مختلفة من النظام. تشمل الاختبارات إدارة المستخدمين والعقارات والشركات والوسطاء والمدن، إضافة إلى التوصيات، الاستثمار، المحافظ، الإشعارات، الروابط الاجتماعية، الثقة، التقييمات، وطلبات التوثيق.

تساعد الاختبارات على التأكد من أن المسارات والخدمات تعمل كما هو متوقع، وأن القيود المهمة مثل منع الوصول غير المصرح به أو منع البيانات المكررة لا تزال فعالة بعد التعديلات.

## 8.11. أوامر التشغيل والبناء

لتشغيل الخادم الخلفي:

```bash
cd project-RealEstate_database
php artisan serve
```

لتشغيل الواجهة الأمامية:

```bash
cd project-RealEstate
npm run dev
```

لبناء الواجهة للإنتاج:

```bash
cd project-RealEstate
npm run build
```

لتشغيل الاختبارات:

```bash
cd project-RealEstate_database
php artisan test
```

لتشغيل خدمة التنبؤ السعري:

```bash
cd project-RealEstate_database/ml/pricing
python server.py
```

ومن الأوامر المفيدة في Laravel:

```bash
php artisan migrate
php artisan db:seed
php artisan optimize:clear
```

## 8.13. خلاصة الفصل

استخدم المشروع مجموعة أدوات مناسبة لبناء منصة عقارية حديثة. فقد تم استخدام Laravel لتوفير خادم API قوي، وVue لبناء واجهة تفاعلية، وFlask لتقديم خدمة تعلم آلي مستقلة، إضافة إلى أدوات مساندة للاختبار، التوثيق، الخرائط، وإدارة الحزم.

يساعد هذا الاختيار على فصل أجزاء النظام وتطوير كل جزء بشكل مستقل. كما يجعل المشروع قابلاً للتوسع مستقبلاً بإضافة خدمات جديدة مثل الدفع الإلكتروني، تحسين التوصيات، أو تطوير خدمة التنبؤ السعري دون الحاجة إلى إعادة بناء النظام بالكامل.
