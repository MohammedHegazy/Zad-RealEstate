# الفصــل التاسع

# دليل الاستخدام

## 9.1. مقدمة

يوضح هذا الفصل طريقة استخدام منصة زاد للعقارات من وجهة نظر المستخدم النهائي. يركز الدليل على أهم الأدوار داخل النظام: الزائر، المستخدم المسجل، المالك، الشركة، المستثمر، ومدير النظام. كما يوضح الخدمات الرئيسية مثل تصفح العقارات، البحث على الخريطة، المفضلة، التوصيات، الرسائل، الاستثمار، التنبؤ السعري، والتقييمات.

يعتمد النظام على واجهة ويب أمامية تتصل بخادم Laravel API. لذلك يجب تشغيل الواجهة والخادم الخلفي قبل استخدام النظام، وتشغيل خدمة Flask عند الحاجة إلى ميزة التنبؤ بسعر العقار.

## 9.2. الأدوات اللازمة

قبل البدء باستخدام النظام محلياً يجب التأكد من توفر المتطلبات التالية:

- تشغيل مشروع Laravel داخل `project-RealEstate_database`.
- تشغيل مشروع Vue داخل `project-RealEstate`.
- ضبط ملف `.env` الخاص بالخلفية وقاعدة البيانات.
- تثبيت حزم PHP باستخدام Composer.
- تثبيت حزم الواجهة باستخدام npm.
- تشغيل خدمة Flask الخاصة بالتنبؤ السعري عند الحاجة.
- تنفيذ `storage:link` إذا كانت الصور والملفات العامة تحتاج إلى عرض من التخزين.

لتشغيل الخادم الخلفي:

```bash
cd project-RealEstate_database
php artisan serve
```

لتشغيل الواجهة الأمامية:

```bash
cd project-RealEstate
npm run dev
```

لتشغيل خدمة التنبؤ السعري:

```bash
cd project-RealEstate_database/ml/pricing
python server.py
```

## 9.3. الدخول إلى النظام وإنشاء الحساب

يستطيع الزائر تصفح أجزاء عامة من المنصة دون تسجيل دخول، مثل العقارات والمدن والشركات والوسطاء. أما الوظائف الخاصة مثل المفضلة، الرسائل، التوصيات، التقييمات، التحليلات، والمحافظ فتحتاج إلى حساب مستخدم.

لإنشاء حساب جديد، يفتح المستخدم صفحة التسجيل ويدخل البيانات المطلوبة مثل الاسم والبريد الإلكتروني وكلمة المرور. بعد نجاح التسجيل يمكنه تسجيل الدخول إلى النظام والوصول إلى الصفحات المحمية.

عند تسجيل الدخول، يتحقق النظام من البيانات عبر Laravel Sanctum، ثم يسمح للمستخدم باستخدام الخدمات الخاصة. وعند تسجيل الخروج يتم إنهاء الجلسة الحالية ومنع استخدام المسارات المحمية حتى يتم تسجيل الدخول مرة أخرى.

## 9.4. استخدام الزائر

يستطيع الزائر استخدام المنصة في وضع التصفح العام. وتشمل أهم العمليات المتاحة له:

- فتح الصفحة الرئيسية والتعرف على محتوى المنصة.
- تصفح قائمة العقارات المتاحة.
- فتح صفحة تفاصيل العقار ومشاهدة المعلومات الأساسية.
- عرض صور وفيديوهات العقار عند توفرها.
- تصفح المدن والمناطق.
- تصفح الشركات العقارية.
- تصفح الوسطاء العقاريين.
- استخدام الخريطة لمشاهدة مواقع العقارات.

إذا حاول الزائر استخدام وظيفة خاصة مثل المفضلة أو الرسائل، فيجب أن يوجهه النظام إلى تسجيل الدخول.

## 9.5. استخدام المستخدم المسجل

بعد تسجيل الدخول يستطيع المستخدم الوصول إلى وظائف أوسع من الزائر. يمكنه تعديل ملفه الشخصي وإدارة معلوماته الأساسية وروابط التواصل الخاصة به. كما يستطيع حفظ العقارات أو الوسطاء في المفضلة للرجوع إليها لاحقاً.

يمكن للمستخدم ضبط تفضيلاته مثل المدينة، نوع العقار، حدود السعر، أو الاهتمامات العامة. يستخدم النظام هذه التفضيلات مع تفاعلات المستخدم لتوليد توصيات عقارية مناسبة.

كما يستطيع المستخدم إرسال الرسائل إلى الجهات المرتبطة بالعقارات، متابعة الإشعارات، كتابة تقييمات للعقارات أو الشركات أو الوسطاء، وطلب توقع سعر عقار عند توفر الخدمة.

أهم خدمات المستخدم المسجل:

| الخدمة | طريقة الاستخدام |
|---|---|
| الملف الشخصي | فتح صفحة الحساب وتعديل البيانات. |
| المفضلة | الضغط على زر المفضلة داخل بطاقة العقار أو الوسيط. |
| التوصيات | ضبط التفضيلات ثم فتح صفحة التوصيات. |
| الرسائل | فتح صفحة الرسائل أو إرسال رسالة من تفاصيل العقار. |
| الإشعارات | فتح صفحة الإشعارات ومتابعة الحالات الجديدة. |
| التقييمات | كتابة تقييم من صفحة العقار أو الجهة المناسبة. |

## 9.6. استخدام مالك العقار

يستطيع مالك العقار إضافة عقار جديد وإدارة بياناته حسب الصلاحيات المتاحة. يدخل المالك معلومات العقار مثل العنوان، النوع، السعر، المدينة، المنطقة، الوصف، والإحداثيات الجغرافية.

بعد إدخال البيانات الأساسية، يمكن رفع صور وفيديوهات للعقار. هذه الوسائط تساعد المستخدمين على فهم العقار بشكل أوضح قبل التواصل أو اتخاذ قرار. ويمكن للمالك لاحقاً تعديل بيانات العقار أو تحديث وسائطه.

قد يحتاج العقار إلى مراجعة أو تغيير حالة من قبل الإدارة قبل ظهوره بشكل كامل للمستخدمين، حسب سياسة النظام.

## 9.7. استخدام الشركة والوسيط

تستطيع الشركة العقارية إدارة ملفها الخاص، مثل اسم الشركة، وصفها، بيانات التواصل، الروابط الاجتماعية، والمنطقة المرتبطة بها. كما يمكن ربط الوسطاء بالشركة حتى يظهروا ضمن ملفها أو ضمن صفحات الوسطاء.

أما الوسيط فيملك ملفاً يعرض بياناته وتقييماته ودرجة الثقة المرتبطة به. يستطيع المستخدمون تصفح الوسطاء، مشاهدة معلوماتهم، وربما التواصل معهم من خلال الرسائل أو بيانات التواصل المتاحة.

وجود ملفات الشركات والوسطاء يساعد على تنظيم الجهات العقارية داخل المنصة، ويمنح المستخدم طريقة أوضح لمعرفة الجهة المسؤولة عن العقار.

## 9.8. استخدام المستثمر

يستفيد المستثمر من الوظائف التحليلية في المنصة. يمكنه فتح خدمة التحليل الاستثماري وإدخال بيانات مرتبطة بالعقار والتكاليف والدخل المتوقع. يعرض النظام مؤشرات تساعد على تقييم العقار من زاوية مالية، مثل العائد أو فترة الاسترداد حسب البيانات المتوفرة.

كما يستطيع المستثمر إنشاء محفظة عقارية وإضافة عقارات إليها. تفيد المحفظة في متابعة مجموعة عقارات معاً بدلاً من متابعة كل عقار بشكل منفصل. وتساعد لوحة المستثمر على عرض ملخصات أو مؤشرات مرتبطة بالمحافظ والتحليلات.

خطوات الاستخدام المختصرة:

1. اختيار عقار مناسب.
2. فتح التحليل الاستثماري.
3. إدخال البيانات المطلوبة.
4. مراجعة النتائج.
5. إنشاء محفظة أو اختيار محفظة موجودة.
6. إضافة العقار إلى المحفظة.

## 9.9. خدمة التنبؤ بسعر العقار

تسمح خدمة التنبؤ السعري بالحصول على سعر تقديري للعقار اعتماداً على بياناته. عند طلب التنبؤ، يرسل النظام بيانات العقار إلى Laravel، ثم يرسل Laravel البيانات إلى خدمة Flask، وبعدها تعود النتيجة ليتم عرضها في الواجهة.

لا تعمل هذه الخدمة إلا إذا كانت خدمة Flask الخاصة بالتنبؤ السعري قيد التشغيل. كما يجب أن تكون بيانات العقار كافية حتى يستطيع النموذج إنتاج نتيجة مفيدة.

## 9.10. استخدام المدير

يمتلك مدير النظام صلاحيات أوسع من باقي المستخدمين. بعد تسجيل الدخول بحساب مدير، يستطيع الدخول إلى لوحة الإدارة ومتابعة أجزاء النظام المختلفة.

تشمل مهام المدير:

- إدارة المستخدمين وحساباتهم.
- إدارة العقارات وتغيير حالاتها.
- إدارة المدن والمناطق.
- إدارة الشركات العقارية.
- إدارة الوسطاء.
- مراجعة التقييمات.
- مراجعة طلبات التوثيق.
- متابعة الرسائل والإشعارات عند الحاجة.
- عرض إحصاءات لوحة التحكم.
- إدارة الروابط أو البيانات المرتبطة بالمحتوى.

تهدف لوحة الإدارة إلى الحفاظ على جودة البيانات ومنع ظهور محتوى غير مناسب أو غير موثوق.

## 9.11. ملاحظات تشغيلية

عند استخدام النظام محلياً يجب الانتباه إلى النقاط التالية:

- يجب تشغيل Laravel قبل استخدام الواجهة.
- يجب ضبط رابط API في إعدادات الواجهة حتى تصل الطلبات إلى الخادم الصحيح.
- يجب تنفيذ الهجرات وملء البيانات الأولية عند الحاجة.
- يجب ضبط صلاحية المدير لاستخدام لوحة الإدارة.
- يجب تشغيل خدمة Flask قبل استخدام التنبؤ السعري.
- يجب ضبط إحداثيات العقارات حتى تظهر بشكل صحيح على الخريطة.
- يجب تشغيل رابط التخزين إذا كانت الصور لا تظهر.
- بعض الوظائف تحتاج إلى تسجيل دخول، لذلك لا تعمل من حساب زائر.

## 9.12. خلاصة الفصل

يوفر النظام تجربة مختلفة حسب نوع المستخدم. فالزائر يستطيع التصفح، والمستخدم المسجل يستطيع التفاعل والحفظ والمراسلة والتقييم، والمالك يستطيع إدارة عقاراته، والشركة والوسيط يستطيعان عرض ملفاتهما، والمستثمر يستطيع استخدام التحليلات والمحافظ، بينما يدير المدير النظام من لوحة التحكم.

بهذا يغطي دليل الاستخدام المسارات الأساسية التي يحتاجها المستخدم للتعامل مع المنصة، مع توضيح المتطلبات التشغيلية والخدمات الرئيسية التي يقدمها مشروع زاد للعقارات.
