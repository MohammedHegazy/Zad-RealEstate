# الفصــل الأوَّل

# التعريف بالمشروع

## 1.1. المقدمة

زاد للعقارات هو مشروع منصة ويب عربية تهدف إلى تنظيم عملية البحث عن العقارات والتفاعل معها. يوفر النظام بيئة موحدة لعرض العقارات والمدن والمناطق والشركات والوسطاء، مع دعم وظائف إضافية مثل المفضلة، التوصيات، الرسائل، التحليل الاستثماري، المحافظ العقارية، التقييمات، التوثيق، والتنبؤ السعري.

يعتمد المشروع على واجهة أمامية مبنية باستخدام Vue، وخادم خلفي مبني باستخدام Laravel، إضافة إلى خدمة مستقلة للتنبؤ بسعر العقار باستخدام Flask وscikit-learn. هذا التقسيم يجعل النظام قابلاً للتطوير، لأن كل طبقة لها مسؤولية واضحة.

## 1.2. تموضع المشروع في سياق المشاريع الأخرى وأهميته

ينتمي المشروع إلى مجال المنصات العقارية الرقمية، لكنه لا يركز فقط على عرض الإعلانات العقارية. فالكثير من الأنظمة العقارية تقدم قوائم عقارات وصوراً ومعلومات عامة، بينما يحاول هذا المشروع إضافة وظائف مساعدة على اتخاذ القرار، مثل التوصيات، التحليل الاستثماري، درجة الثقة، وطلب توقع السعر.

تظهر أهمية المشروع من طبيعة السوق العقاري نفسه. فاختيار العقار يحتاج إلى معلومات دقيقة عن الموقع والسعر والجهة المالكة أو الوسيط، كما يحتاج المستثمر إلى مؤشرات تساعده على تقييم الجدوى. لذلك يحاول النظام جمع بيانات العرض والتفاعل والتحليل داخل منصة واحدة.

## 1.3. مبررات المشروع

تم بناء المشروع استجابة لمجموعة من المشكلات التي تظهر عند البحث عن العقارات أو إدارتها:

- تشتت بيانات العقارات بين أكثر من مصدر وعدم وجود تنظيم واضح لها.
- صعوبة البحث حسب المدينة أو المنطقة أو الموقع الجغرافي.
- الحاجة إلى عرض الشركات والوسطاء بطريقة منظمة ومرتبطة بالعقارات.
- الحاجة إلى حفظ العقارات المهمة في المفضلة للرجوع إليها لاحقاً.
- الحاجة إلى توصيات مخصصة تعتمد على تفضيلات المستخدم وتفاعلاته.
- ضعف الثقة في بعض الإعلانات العقارية وضرورة وجود تقييمات وتوثيق.
- حاجة المستثمر إلى أدوات تحليل تساعده على مقارنة العقارات.
- الحاجة إلى تواصل منظم عبر الرسائل والإشعارات.
- الحاجة إلى لوحة إدارة لمراجعة المستخدمين والمحتوى والطلبات.
- الحاجة إلى تقدير سعري مساعد يعتمد على بيانات العقار.

هذه المبررات توضح أن المشكلة ليست في عرض العقار فقط، بل في تنظيم رحلة المستخدم كاملة من البحث حتى التقييم والتواصل واتخاذ القرار.

## 1.4. أهداف المشروع

يهدف المشروع إلى تحقيق مجموعة من الأهداف العملية:

| الهدف | التوضيح |
|---|---|
| بناء منصة عقارية عربية | توفير واجهة واضحة لتصفح العقارات والجهات العقارية. |
| تنظيم بيانات العقارات | ربط العقارات بالمدن والمناطق والوسائط والملاك أو الجهات. |
| تحسين البحث | دعم التصفية والبحث الجغرافي وعرض العقارات على الخريطة. |
| دعم المستخدم المسجل | توفير المفضلة، الرسائل، الإشعارات، التفضيلات، والتقييمات. |
| دعم المستثمر | توفير التحليل الاستثماري والمحافظ العقارية. |
| تعزيز الثقة | دعم المراجعات وطلبات التوثيق ودرجة الثقة. |
| دعم الإدارة | توفير لوحة لإدارة المستخدمين والعقارات والشركات والوسطاء. |
| إضافة تحليل ذكي | ربط النظام بخدمة توقع سعر العقار. |

## 1.5. ميزات المنصة الإلكترونية

تحتوي المنصة على مجموعة ميزات رئيسية، أهمها:

### إدارة الحسابات والمصادقة

يدعم النظام إنشاء الحساب وتسجيل الدخول والخروج، مع حماية المسارات الخاصة باستخدام Laravel Sanctum. وبذلك يستطيع المستخدم الوصول إلى وظائف لا تظهر للزائر مثل المفضلة والرسائل والتوصيات.

### تصفح العقارات

يمكن للمستخدم تصفح العقارات، فتح صفحة التفاصيل، مشاهدة الصور والفيديوهات، قراءة الوصف، معرفة السعر والموقع، والاطلاع على بيانات الجهة المرتبطة بالعقار.

### المدن والمناطق والخريطة

يربط النظام العقارات بالمدن والمناطق، كما يدعم عرض مواقع العقارات على الخريطة باستخدام Leaflet وOpenStreetMap. يساعد ذلك المستخدم على فهم موقع العقار قبل اتخاذ القرار.

### الشركات والوسطاء

يعرض النظام الشركات العقارية والوسطاء بملفات واضحة، مع بيانات التواصل والروابط والتقييمات ودرجة الثقة. هذا يضيف تنظيماً للجهات المسؤولة عن العقارات.

### المفضلة والتوصيات

يستطيع المستخدم حفظ العقارات أو الوسطاء في المفضلة. كما يستطيع ضبط تفضيلاته ليحصل على توصيات أقرب لاهتماماته، اعتماداً على البيانات التي يخزنها النظام عن التفضيلات والتفاعلات.

### الاستثمار والمحافظ

يدعم النظام إنشاء تحليلات استثمارية لعقارات محددة، كما يسمح بإنشاء محافظ عقارية وإضافة عقارات إليها. هذه الميزة موجهة للمستخدم الذي ينظر إلى العقار كفرصة استثمارية.

### الرسائل والإشعارات

يوفر النظام رسائل للتواصل بين المستخدمين أو الجهات المرتبطة بالعقار، بالإضافة إلى إشعارات تساعد المستخدم على متابعة الأحداث المهمة.

### التقييمات والثقة والتوثيق

يدعم النظام تقييم العقارات والوسطاء والشركات، كما يوفر طلبات توثيق ودرجة ثقة. تساعد هذه الوظائف على رفع موثوقية البيانات وتحسين قرار المستخدم.

### لوحة الإدارة والتنبؤ السعري

تسمح لوحة الإدارة بمتابعة المستخدمين والعقارات والشركات والوسطاء والتقييمات والتوثيق. كما يدعم النظام خدمة تنبؤ سعري منفصلة ترجع سعراً تقديرياً للعقار بناءً على بياناته.

## 1.6. منهجية العمل

اعتمد العمل على منهجية تدريجية تبدأ بدراسة المشكلة والأنظمة المشابهة، ثم تحليل المتطلبات وحالات الاستخدام، وبعدها تصميم قاعدة البيانات والمخططات، ثم تنفيذ الواجهة الخلفية والواجهة الأمامية وخدمة التنبؤ السعري، وأخيراً إعداد الاختبارات والتوثيق.

ساعد هذا الأسلوب على تقسيم المشروع إلى وحدات واضحة، مثل المصادقة، العقارات، الشركات، التوصيات، الاستثمار، الرسائل، الثقة، والإدارة. وبذلك يمكن تطوير كل وحدة ومراجعتها دون فقدان صورة النظام العامة.

## 1.7. شرائح المستخدمين

يخدم النظام عدة شرائح:

| الشريحة | الوصف |
|---|---|
| الزائر | يتصفح العقارات والمدن والشركات والوسطاء دون تسجيل دخول. |
| المشتري أو الباحث | يبحث عن عقار مناسب ويحفظ المفضلة ويتواصل مع الجهات. |
| المستثمر | يستخدم التحليل الاستثماري والمحافظ لمقارنة العقارات. |
| المالك | يدير عقاراته ووسائطها وبياناتها. |
| الشركة العقارية | تعرض ملفها وتدير بياناتها ووسطاءها. |
| الوسيط العقاري | يظهر بملف خاص ويمكن تقييمه والتواصل معه. |
| مدير النظام | يدير المستخدمين والمحتوى والثقة والتوثيق والحالات. |

هذا التنوع في المستخدمين يوضح أن المشروع ليس موجهاً لزائر واحد فقط، بل يحاول تنظيم العلاقة بين أطراف السوق العقاري.

## 1.8. القيمة المضافة

القيمة الأساسية التي يقدمها المشروع هي جمع العرض العقاري، التفاعل، الثقة، والتحليل في منصة واحدة. فالمستخدم لا يكتفي بمشاهدة إعلان، بل يستطيع معرفة موقع العقار، حفظه، مقارنة الخيارات، التواصل، قراءة التقييمات، والحصول على توصيات.

كما يقدم المشروع قيمة للجهات العقارية من خلال ملفات الشركات والوسطاء، وللمستثمر من خلال التحليل والمحافظ، وللمدير من خلال لوحة إدارة تسمح بضبط المحتوى ومراجعة التوثيق والتقييمات.

يمكن تلخيص القيمة المضافة في النقاط التالية:

- تقليل تشتت البيانات العقارية.
- تحسين تجربة البحث والمقارنة.
- دعم القرار الاستثماري.
- رفع مستوى الثقة من خلال التقييمات والتوثيق.
- تنظيم التواصل داخل المنصة.
- توفير أساس قابل للتوسع مستقبلاً.

## 1.9. محتويات الفصول اللاحقة

تعرض الفصول التالية الدراسة المرجعية للأنظمة المشابهة، ثم الدراسة التحليلية التي توضح المتطلبات وحالات الاستخدام، وبعدها الدراسة التصميمية التي تشرح قاعدة البيانات والمخططات والبنية المعمارية. ثم يتم عرض إدارة المشروع، التنفيذ، حالات الاختبار، الأدوات واللغات المستخدمة، دليل الاستخدام، وأخيراً الخلاصة والآفاق المستقبلية.
