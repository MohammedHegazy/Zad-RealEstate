# الفصــل التاسع

# دليل الاستخدام

## 9.1. مقدمة

يهدف هذا الفصل إلى توضيح طريقة استخدام منصة **زاد للعقارات** من وجهة نظر المستخدم النهائي. تم تصميم النظام ليخدم أكثر من نوع مستخدم، لذلك تختلف طريقة الاستخدام حسب الدور: الزائر، المستخدم المسجل، المشتري أو المستثمر، مالك العقار، الشركة العقارية، الوسيط، ومدير النظام.

يعرض هذا الدليل الخطوات الأساسية للتعامل مع النظام، بدءاً من تشغيل المشروع محلياً، ثم إنشاء الحساب وتسجيل الدخول، ثم استخدام الخدمات الرئيسية مثل تصفح العقارات، البحث على الخريطة، المفضلة، التوصيات، التحليل الاستثماري، إدارة العقارات، إدارة الشركات والوسطاء، التقييمات والثقة، والتنبؤ السعري.

## 9.2. الأدوات اللازمة

لاستخدام النظام محلياً يحتاج المستخدم أو المطور إلى تشغيل أكثر من جزء:

| الجزء | المطلوب |
|---|---|
| الخادم الخلفي | PHP، Composer، Laravel، قاعدة بيانات |
| الواجهة الأمامية | Node.js، npm، Vite |
| خدمة التنبؤ السعري | Python، pip، Flask |
| قاعدة البيانات | MySQL أو SQLite |
| المتصفح | متصفح حديث يدعم تطبيقات Vue |

### تشغيل الخادم الخلفي

```bash
cd project-RealEstate_database
composer install
cp .env.example .env
php artisan key:generate
php artisan migrate --seed
php artisan serve
```

### تشغيل الواجهة الأمامية

```bash
cd project-RealEstate
npm install
npm run dev
```

### تشغيل خدمة التنبؤ السعري

```bash
cd project-RealEstate_database/ml/pricing
pip install -r requirements.txt
python server.py
```

لا يحتاج المستخدم العادي إلى تشغيل خدمة Flask إذا كان يريد تجربة التصفح والإدارة فقط، لكنها مطلوبة عند استخدام ميزة التنبؤ بسعر العقار.

## 9.3. الدخول إلى النظام وإنشاء الحساب

### 9.3.1. إنشاء حساب جديد

لإنشاء حساب جديد، ينتقل المستخدم إلى صفحة التسجيل. يقوم بإدخال بياناته الأساسية مثل اسم المستخدم، الاسم الأول، الاسم الأخير، البريد الإلكتروني، كلمة المرور، رقم الهاتف، رمز الدولة، الجنس، ونوع الحساب. بعد إرسال النموذج، يتحقق النظام من صحة البيانات، ثم ينشئ الحساب ويعيد توكن تسجيل الدخول.

بعد إنشاء الحساب يستطيع المستخدم الوصول إلى الوظائف المحمية مثل المفضلة، التوصيات، الرسائل، التحليلات الاستثمارية، وطلبات التوثيق. أما الوظائف العامة مثل تصفح العقارات والمدن والمناطق والشركات والوسطاء فيمكن استخدامها حتى قبل إنشاء الحساب.

### 9.3.2. تسجيل الدخول

يدخل المستخدم بريده الإلكتروني وكلمة المرور في صفحة تسجيل الدخول. إذا كانت البيانات صحيحة وكان الحساب فعالاً، ينشئ النظام توكن مصادقة ويحول المستخدم إلى الواجهة المناسبة. أما إذا كانت البيانات غير صحيحة، تظهر رسالة خطأ ولا يتم إنشاء جلسة.

إذا كان المستخدم مديراً، يمكنه الدخول إلى لوحة الإدارة. أما المستخدم العادي فيستخدم الواجهة العامة والصفحات المحمية مثل الملف الشخصي والمفضلة والتوصيات.

### 9.3.3. تسجيل الخروج

عند تسجيل الخروج، يحذف النظام التوكن الحالي للمستخدم. بعد ذلك لا يستطيع المستخدم الوصول إلى الوظائف المحمية إلا بعد تسجيل الدخول من جديد.

## 9.4. تعريف الخدمات الرئيسية

### 9.4.1. خدمة تصفح العقارات

تعد خدمة تصفح العقارات من أهم خدمات النظام. يستطيع الزائر أو المستخدم المسجل الدخول إلى صفحة العقارات ومراجعة قائمة العقارات المنشورة. تعرض القائمة بيانات مختصرة تساعد على المقارنة الأولية مثل اسم العقار، السعر، الموقع، وبعض المواصفات.

عند فتح صفحة تفاصيل العقار، تظهر بيانات أوسع تشمل:

- اسم العقار ووصفه.
- المدينة والمنطقة.
- السعر وسعر المتر.
- المساحة وعدد الغرف والمرافق.
- حالة البناء وتاريخ البناء عند توفره.
- الصور والفيديوهات والإعلانات.
- الموقع الجغرافي.
- معلومات الاستثمار مثل ROI وفترة الاسترداد عند توفرها.
- التقييمات ودرجة الثقة المرتبطة بالجهة.

إذا كان المستخدم مسجلاً، يستطيع تنفيذ إجراءات إضافية مثل حفظ العقار في المفضلة، إرسال تفاعل، إنشاء تقييم، أو طلب توقع السعر.

خطوات الاستخدام:

1. افتح صفحة العقارات من القائمة الرئيسية.
2. راجع البطاقات أو الصفوف المعروضة.
3. استخدم البحث أو الفلاتر عند توفرها لتقليل النتائج.
4. اضغط على العقار المناسب لفتح التفاصيل.
5. راجع الصور والمعلومات المالية والموقع قبل اتخاذ قرار.
6. إذا كنت مسجلاً، استخدم المفضلة أو التقييم أو التنبؤ السعري حسب الحاجة.

### 9.4.2. خدمة البحث على الخريطة

تسمح صفحة الخريطة بعرض العقارات حسب الإحداثيات. يدخل المستخدم إلى صفحة `البحث على الخريطة`، فيعرض النظام علامات للعقارات المتاحة. يمكن للمستخدم تحريك الخريطة أو اختيار عقار لفتح تفاصيله.

تعتمد هذه الخدمة على توفر إحداثيات صحيحة للعقارات والمدن والمناطق. إذا لم تكن الإحداثيات موجودة أو صحيحة، قد لا يظهر العقار على الخريطة بالشكل المتوقع.

خطوات الاستخدام:

1. افتح صفحة الخريطة من قسم العقارات.
2. انتظر تحميل العلامات على الخريطة.
3. قرّب أو أبعد الخريطة حسب المنطقة المطلوبة.
4. اضغط على علامة عقار لعرض بيانات مختصرة.
5. انتقل إلى تفاصيل العقار إذا كان مناسباً.

### 9.4.3. خدمة تصفح المدن والمناطق

توفر المنصة صفحات للمدن والمناطق. يستطيع المستخدم فتح قائمة المدن، ثم فتح تفاصيل مدينة لعرض المناطق أو العقارات المرتبطة بها. كما يستطيع فتح قائمة المناطق أو صفحة تفاصيل منطقة. تساعد هذه الخدمة على البحث العقاري حسب الموقع بدلاً من البحث العام فقط.

### 9.4.4. خدمة تصفح الشركات والوسطاء

يستطيع المستخدم تصفح دليل الشركات العقارية ودليل الوسطاء. عند فتح صفحة شركة، تظهر بياناتها مثل الاسم والوصف والمنطقة وروابط التواصل والوسطاء التابعين لها وتقييماتها ودرجة الثقة. وعند فتح صفحة وسيط، تظهر بيانات الوسيط وارتباطه بالشركة وتقييماته ودرجة ثقته.

هذه الخدمة تساعد المستخدم على معرفة الجهة العارضة أو الوسيط قبل التواصل أو الاعتماد على الإعلان.

### 9.4.5. خدمة المفضلة

بعد تسجيل الدخول، يستطيع المستخدم إضافة العقارات إلى المفضلة للرجوع إليها لاحقاً. كما يستطيع حفظ الوسطاء في المفضلة عند توفر الزر الخاص بذلك. تظهر العناصر المحفوظة في صفحة المفضلة، ويمكن للمستخدم إزالة أي عنصر لم يعد مهتماً به.

تستخدم المفضلة أيضاً كإشارة مهمة في نظام التوصيات، لأن حفظ عقار أو وسيط يدل على اهتمام المستخدم بنوع أو موقع أو جهة معينة.

خطوات الاستخدام:

1. سجل الدخول إلى حسابك.
2. افتح تفاصيل عقار أو وسيط.
3. اضغط على زر الإضافة إلى المفضلة.
4. افتح صفحة المفضلة لمراجعة العناصر المحفوظة.
5. أزل العنصر من المفضلة إذا لم يعد مناسباً.

ملاحظة: لا يسمح النظام بتكرار نفس العقار أو الوسيط في مفضلة المستخدم نفسه.

### 9.4.6. خدمة التفضيلات والتوصيات

يستطيع المستخدم المسجل ضبط تفضيلاته العقارية مثل المدينة، المنطقة، الميزانية الدنيا والعليا، نوع العقار، عدد الغرف، هدف الاستثمار، ومستوى المخاطرة. بعد حفظ التفضيلات، يمكنه فتح صفحة التوصيات لعرض عقارات مقترحة.

يعتمد النظام على التفضيلات والمفضلة والتفاعلات لتوليد التوصيات. إذا لم يضف المستخدم أي تفضيلات أو مفضلات، قد تظهر رسالة تطلب منه إضافة إشارات حتى تصبح التوصيات أكثر دقة. يستطيع المستخدم أيضاً تحديث التوصيات يدوياً.

خطوات الاستخدام:

1. سجل الدخول.
2. افتح صفحة التوصيات أو إعدادات التفضيلات.
3. حدد الميزانية والموقع والنوع والهدف الاستثماري.
4. احفظ التفضيلات.
5. افتح قائمة التوصيات.
6. راجع أسباب التوصية أو درجة الملاءمة عند توفرها.
7. استخدم زر التحديث إذا أردت إعادة توليد النتائج بعد تغيير المفضلة أو التفضيلات.

### 9.4.7. خدمة التنبؤ بسعر العقار

عند فتح تفاصيل عقار، يمكن للمستخدم المسجل طلب توقع سعر العقار إذا كانت خدمة Flask تعمل. يرسل النظام خصائص العقار إلى خدمة التنبؤ، ثم يعيد السعر المتوقع والفرق بين السعر المتوقع والسعر المعروض.

هذه الخدمة لا تعني أن السعر المتوقع هو السعر النهائي، لكنها تساعد المستخدم على مقارنة السعر المعروض مع مؤشر حسابي مبني على نموذج تعلم آلي. إذا كانت خدمة Flask غير مشغلة، تظهر رسالة خطأ أو عدم توفر الخدمة.

خطوات الاستخدام:

1. سجل الدخول.
2. افتح تفاصيل عقار.
3. اضغط على خيار توقع السعر.
4. انتظر استجابة خدمة التنبؤ.
5. راجع السعر المتوقع والفرق عن السعر المعروض.
6. استخدم النتيجة كمؤشر مساعد فقط، وليس كقرار نهائي.

### 9.4.8. خدمة التحليل الاستثماري

تساعد هذه الخدمة المستخدم على دراسة العقار من ناحية مالية. يستطيع المستخدم إنشاء تحليل استثماري يعتمد على بيانات العقار أو على قيم يدخلها يدوياً، مثل الإيجار الشهري، المصاريف السنوية، الصيانة، الضرائب، ونسبة الإشغال.

بعد حفظ التحليل يعرض النظام مؤشرات مثل:

- الدخل السنوي المتوقع.
- العائد على الاستثمار ROI.
- فترة استرداد رأس المال.

تساعد هذه المؤشرات المستخدم المستثمر على مقارنة أكثر من عقار بطريقة أكثر وضوحاً.

خطوات الاستخدام:

1. افتح عقاراً ترغب بدراسته.
2. اختر إنشاء تحليل استثماري.
3. راجع القيم المأخوذة من العقار أو عدلها.
4. أدخل الإيجار والمصاريف ونسبة الإشغال عند الحاجة.
5. احفظ التحليل.
6. راجع ROI وفترة الاسترداد والدخل المتوقع.

### 9.4.9. خدمة المحافظ الاستثمارية

يستطيع المستخدم إنشاء محفظة استثمارية وإضافة عقارات إليها. عند إضافة عقار إلى المحفظة، يبدأ عادة بحالة `tracking`. يمكن لاحقاً تغيير الحالة إلى `invested` أو `sold` حسب مسار الاستثمار.

تساعد المحافظ على تنظيم العقارات التي يريد المستخدم متابعتها كفرص استثمارية، بدلاً من حفظها فقط في المفضلة. كما يستطيع المستخدم فتح لوحة المستثمر لعرض ملخص عام عن محافظه والعقارات المرتبطة بها.

خطوات الاستخدام:

1. سجل الدخول.
2. افتح قسم المحافظ أو لوحة المستثمر.
3. أنشئ محفظة باسم واضح.
4. افتح عقاراً نشطاً وأضفه إلى المحفظة.
5. تابع العقار داخل المحفظة.
6. غيّر حالته عند الاستثمار أو البيع.
7. افتح لوحة المستثمر لمراجعة الملخص.

### 9.4.10. خدمة الرسائل

تسمح خدمة الرسائل للمستخدم المسجل بالتواصل مع مستخدمين آخرين داخل المنصة. يستطيع المستخدم إرسال رسالة، فتح محادثة، عرض الرسائل، وتحديدها كمقروءة. عند فتح محادثة، يتم تحديث حالة الرسائل الواردة غير المقروءة.

هذه الخدمة تساعد على تنظيم التواصل بدلاً من الاعتماد الكامل على قنوات خارجية. ومع ذلك يمكن للكيانات أيضاً استخدام روابط التواصل الاجتماعية مثل واتساب أو موقع الويب عند توفرها.

خطوات الاستخدام:

1. سجل الدخول.
2. افتح صفحة مستخدم أو جهة تريد التواصل معها عند توفر خيار الرسالة.
3. اكتب نص الرسالة.
4. أرسل الرسالة.
5. افتح المحادثة لاحقاً لمتابعة الردود.
6. عند قراءة المحادثة، يتم تحديث حالة الرسائل الواردة.

### 9.4.11. خدمة الإشعارات

تظهر الإشعارات للمستخدم المسجل لتنبيهه إلى أحداث أو رسائل أو تحديثات مرتبطة بحسابه. يستطيع المستخدم عرض الإشعارات، معرفة عدد غير المقروء، تحديد إشعار كمقروء، تحديد كل الإشعارات كمقروءة، أو حذف إشعار.

أما المدير فيستطيع إنشاء إشعار وإرساله إلى مستخدم عند الحاجة من لوحة الإدارة.

### 9.4.12. خدمة التقييمات

يستطيع المستخدم المسجل كتابة تقييم لعقار أو وسيط أو شركة. لا يظهر التقييم مباشرة للعامة، بل يخضع للمراجعة الإدارية. بعد اعتماد التقييم من المدير، يمكن أن يظهر في القائمة العامة وقد يدخل في حساب متوسط التقييم أو درجة الثقة.

إذا حاول المستخدم تكرار تقييم لنفس الكيان ضمن الحالات التي تمنع التكرار، يرفض النظام الطلب.

خطوات الاستخدام:

1. سجل الدخول.
2. افتح صفحة العقار أو الوسيط أو الشركة.
3. اختر إضافة تقييم.
4. أدخل عدد النجوم والتعليق.
5. أرسل التقييم.
6. انتظر مراجعته من الإدارة.

ملاحظة: التقييمات المعتمدة فقط تظهر للعامة وتدخل في الملخصات.

### 9.4.13. خدمة التوثيق ودرجة الثقة

يستطيع المستخدم إرسال طلب توثيق مرفقاً بنوع المستند والملف المطلوب. يراجع المدير الطلب، ثم يقبله أو يرفضه. يؤثر التوثيق على درجة الثقة، خصوصاً للوسطاء والشركات.

تعرض درجة الثقة للمستخدم كمؤشر يساعده على تقدير موثوقية الجهة، لكنها لا تلغي الحاجة إلى مراجعة تفاصيل العقار والتواصل مع الجهة العارضة.

### 9.4.14. خدمة إدارة عقارات المالك

إذا كان المستخدم مالكاً، يستطيع فتح قسم عقاراته وإضافة عقار جديد. يتطلب ذلك إدخال بيانات مثل الاسم، المنطقة، الإحداثيات، السعر، المساحة، الهاتف، النوع، الوصف، وعدد الغرف والمرافق. بعد حفظ العقار يمكن للمالك إدارة الصور والفيديوهات والإعلانات وروابط التواصل.

يبقى العقار مرتبطاً بحالة نشر. قد يكون قيد المراجعة أو نشطاً أو مرفوضاً. لا يستطيع المالك اعتماد عقاره بنفسه، لأن الاعتماد من صلاحيات المدير.

خطوات الاستخدام:

1. سجل الدخول بحساب مالك.
2. افتح قسم عقاراتي.
3. اختر إضافة عقار.
4. أدخل البيانات الأساسية والموقع والسعر.
5. احفظ العقار.
6. أضف الصور والفيديوهات والإعلانات.
7. حدد الصورة الرئيسية عند الحاجة.
8. تابع حالة العقار حتى تتم مراجعته.

### 9.4.15. خدمة إدارة الشركة والوسطاء

إذا كان المستخدم صاحب شركة عقارية، يستطيع إنشاء ملف شركة وتحديثه. يتضمن الملف اسم الشركة، المنطقة، الموقع الإلكتروني، عدد الموظفين، الوصف، أيام العمل، الصور، وروابط التواصل. بعد إنشاء الشركة يمكنه إدارة الوسطاء التابعين لها.

تظهر الشركة للعامة عند اعتمادها حسب حالتها. أما إذا كانت قيد المراجعة أو مرفوضة أو موقوفة، فقد لا تظهر في الدليل العام بنفس الطريقة.

خطوات الاستخدام:

1. سجل الدخول بحساب شركة.
2. افتح ملف شركتي.
3. أدخل بيانات الشركة وأيام العمل.
4. أضف الصور وروابط التواصل.
5. احفظ الملف.
6. افتح قسم الوسطاء.
7. أضف وسيطاً تابعاً للشركة أو حدث بيانات وسيط موجود.
8. تابع حالة اعتماد الشركة.

### 9.4.16. خدمة إدارة النظام من قبل Admin

يدخل مدير النظام إلى لوحة الإدارة لإدارة المنصة. تشمل لوحة الإدارة:

- إحصاءات عامة.
- إدارة المستخدمين.
- إدارة العقارات ووسائطها.
- إدارة الشركات وحالاتها.
- إدارة الوسطاء.
- إدارة المدن والمناطق.
- إدارة الروابط الاجتماعية.
- إدارة الإشعارات.
- مراقبة الرسائل.
- مراجعة التقييمات.
- مراجعة طلبات التوثيق.
- إعادة حساب درجة الثقة عند الحاجة.

تعد لوحة الإدارة مركز التحكم الرئيسي في جودة بيانات المنصة، لأن المدير هو المسؤول عن اعتماد أو رفض المحتوى الحساس.

خطوات الاستخدام:

1. سجل الدخول بحساب مدير.
2. افتح `/admin/dashboard`.
3. راجع الإحصاءات العامة.
4. افتح القسم المطلوب من القائمة الجانبية.
5. استخدم الجداول للبحث والفلترة.
6. افتح تفاصيل السجل عند الحاجة.
7. عدل البيانات أو غيّر الحالة أو احذف السجل حسب الصلاحية.
8. في صفحة الثقة، راجع التقييمات وطلبات التوثيق قبل الاعتماد أو الرفض.

### 9.4.17. خدمة تعديل الملف الشخصي

يستطيع المستخدم المسجل تعديل بيانات ملفه الشخصي من صفحة الحساب. كما يستطيع إدارة بعض روابط التواصل الخاصة به. يجب أن تكون البيانات المدخلة صحيحة، ويعرض النظام أخطاء تحقق إذا كانت البيانات ناقصة أو غير مقبولة.

## 9.5. ملاحظات تشغيلية

- يجب تشغيل Laravel API قبل تشغيل الواجهة حتى تستطيع الواجهة جلب البيانات.
- يجب ضبط `VITE_API_BASE_URL` في الواجهة ليشير إلى مسار API الصحيح.
- يجب تشغيل خدمة Flask عند استخدام التنبؤ السعري فقط.
- يجب ضبط `ML_PRICE_PREDICTION_URL` في الخلفية إذا تغير عنوان خدمة Flask.
- يجب تنفيذ `php artisan storage:link` عند الحاجة إلى عرض الملفات المرفوعة من التخزين العام.
- تعتمد الخريطة على وجود إحداثيات صحيحة للعقارات.
- لا تظهر الشركات غير المعتمدة في الدليل العام بنفس طريقة الشركات المعتمدة.
- لا تظهر التقييمات المعلقة أو المرفوضة للمستخدم العام.
- لا يستطيع المستخدم العادي الوصول إلى مسارات الإدارة حتى لو عرف الرابط.
- في حال ظهور خطأ 401 يجب تسجيل الدخول من جديد.
- في حال ظهور خطأ 403 فهذا يعني أن المستخدم لا يملك الصلاحية المطلوبة.
- في حال ظهور خطأ 422 يجب مراجعة بيانات النموذج لأن هناك حقولاً غير صحيحة.

## 9.6. خلاصة الفصل

يوضح دليل الاستخدام أن منصة زاد للعقارات تقدم تجربة متعددة الأدوار. فالزائر يستطيع التصفح والبحث، والمستخدم المسجل يستطيع حفظ العقارات والتوصيات والرسائل والتقييمات، والمستثمر يستطيع استخدام التحليل والمحافظ، والمالك يستطيع إدارة عقاراته، والشركة تستطيع إدارة ملفها ووسطائها، والمدير يستطيع ضبط المحتوى والثقة من لوحة الإدارة.

وبذلك لا يقتصر استخدام النظام على عرض العقارات فقط، بل يغطي دورة أوسع تشمل الاكتشاف، المقارنة، الحفظ، التواصل، التحليل، التوثيق، والمراجعة الإدارية.
